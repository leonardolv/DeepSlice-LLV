DeepSlice Desktop - Professional Tutorial

This guide mirrors the app exactly and tells you what to click, where to click it, and when each action should be used. Follow it end-to-end to go from raw histology images to curated QuickNII-compatible outputs and reproducible reports.

11 Lessons Beginner to Advanced ~45-60 min
Beginner

Lesson 0 - Prerequisites

  • Windows 10/11 (64-bit)
  • Python 3.10+ in a virtual environment
  • At least 8 GB RAM (16 GB+ recommended for larger datasets)
  • Histology images in .jpg, .jpeg, .png, .tif, or .tiff
  • File naming with section IDs (recommended): _s001, _s002, ...
Important: The current desktop release supports coronal orientation in the main workflow. Sagittal and horizontal are listed as coming soon.
Beginner

Lesson 1 - Installation (Including TensorFlow Fix)

Set up a clean environment, install DeepSlice, and ensure TensorFlow is available so prediction can run.

1

Open PowerShell in the repository root

cd "D:\CLOUD\OneDrive - O365 Turun yliopisto\CODE\GITHUB\DeepSlice-LLV"
2

Create and activate virtual environment

python -m venv .venv
.venv\Scripts\activate
3

Install the project

python -m pip install --upgrade pip
python -m pip install -e .
4

Install critical runtime dependencies

python -m pip install "tensorflow<3.0" scikit-image

This resolves the common startup errors ModuleNotFoundError: tensorflow and ModuleNotFoundError: skimage.

5

Optional: verify import before opening the GUI

python -c "import tensorflow as tf; print(tf.__version__)"
Windows GPU note: TensorFlow on native Windows generally runs CPU-only for newer versions. This is normal; DeepSlice still runs correctly on CPU.

✓ Outcome

The runtime environment is ready and prediction imports no longer fail due to missing TensorFlow.

Lesson 2 - First Launch & Health Check

1

Launch the GUI

Use this immediately after launch to verify whether TensorFlow is in CPU or GPU mode and confirm the runtime is healthy.

3

Confirm error tooling is available

Top bar should include Open Error Log, Copy Last Error, and Try Auto-Fix Last Error.

✓ Outcome

Lesson 3 - UI Orientation

The application uses a left step navigator and a right content stack. Workflow order is strict and intentional.

Step Navigator

Ingestion -> Configuration -> Prediction -> Curation -> Export. Locked steps unlock as data becomes available.

Top Bar

Hardware Health, Save Session, Load Session / QuickNII, Open Error Log, Copy Last Error, Try Auto-Fix Last Error.

Status Prefixes

✓ completed, ● current, ◌ locked/pending.

Runtime Console

Available in Prediction. Use when running alignment or debugging processing tasks.

Workflow rule: If Run Alignment is disabled, check Configuration validation text and Ingestion warnings first; they explain exactly what is blocking execution.
Beginner

Lesson 4 - Ingestion

1

Go to Ingestion (left navigator)

If this is a new session, the app opens here by default.

2

Add images

Click Add Folder for recursive folder import or Add Files for explicit selection. Use Clear to reset.

3

Configure pre-flight options

Leave Detect section numbers from filename (_sXXX) on for indexed workflows. Enable Legacy parser fallback only if your naming does not use _sXXX.

4

Review index and status table

Check for parse errors, duplicates, and missing indices before leaving this step.

5

Thumbnail QA

Click thumbnails to preview loaded files and visually confirm image orientation/quality before inference.

✓ Outcome

Dataset is loaded and pre-flight checks are clear enough to proceed.

Beginner

Lesson 5 - Configuration

1

Select species

Click Mouse (Allen CCFv3) or Rat (Waxholm Rat Atlas) in Species / Atlas. This controls model and atlas configuration.

2

Set physical geometry

Use Auto-estimate thickness for first pass. Click Suggest Thickness when predictions exist and you need a guided value. Use Direction Override only to intentionally override detected direction.

3

Choose prediction mode

Enable Ensemble prediction for robust production runs. Use Use secondary model only for comparison diagnostics, not for normal baseline processing.

4

Read validation banner

The Validation: message in Configuration tells you if execution is ready or what exactly still blocks Run Alignment.

✓ Outcome

Configuration is explicitly validated and ready for prediction.

Beginner

Lesson 6 - Prediction

1

Click Run Alignment

Do this only after Configuration shows ready state. This starts model loading/inference in a background worker.

2

Monitor runtime phase and progress

Use Phase, Progress, and progress bar to track primary/secondary passes and data loading.

3

Open runtime console when needed

Click Show Runtime Console to see detailed logs and troubleshooting context during processing.

4

Accept predicted thickness when available

After completion, click Use Predicted Thickness if you want the estimated value copied into Configuration.

5

Preview per-slice alignment vectors

Use the slice selector to inspect O/U/V overlays and confirm inference output quality before curation.

✓ Outcome

Predictions are generated, indexed direction is detected, and the curation workflow is unlocked.

Intermediate

Lesson 7 - Curation

Use this stage to improve geometric consistency and exclude unreliable sections before export.

1

Flag problematic slices

In the left slice list, check suspicious rows and click Apply Bad Section Flags.

2

Detect outliers automatically (recommended baseline)

Click Detect Outliers to auto-mark low-consistency sections.

3

Apply geometric refinements in this order

  1. Normalize Angles
  2. Enforce Index Order
  3. Enforce Index Spacing

Use this sequence when section numbering is trustworthy and you want stable serial geometry.

4

Manual angle override only when justified

Set ML/DV in spin boxes and click Apply Manual Angles only when you have explicit domain reason to override the inferred mean.

5

Use atlas preview for depth sanity checks

Enable Atlas volume preview, select volume, optionally enable Blend atlas on histology, then adjust blend slider to compare alignment context.

6

Use Undo/Redo aggressively during curation

Every curation action snapshots state. If a transform degrades quality, click Undo immediately.

✓ Outcome

You have a curated, internally consistent dataset ready for publication/export pipelines.

Lesson 8 - Export

1

Set output destination

In Export Configuration, click Browse and choose Output Directory, then set Base Filename.

2

Select primary export format

OptionWhen to use
JSON (QuickNII/VisuAlign)Default for modern workflows and marker preservation.
Legacy XMLUse only for older downstream tooling.

Click Export Predictions

This writes your selected JSON/XML plus an automatic CSV sidecar in the same location.

Generate PDF report

Click Generate Report (PDF) after final curation for a shareable, reproducible summary artifact.

5

Open directly in QuickNII

Set QuickNII path once in this panel, then click Open in QuickNII after a JSON export.

✓ Outcome

You exported reproducible outputs for both computational and visual QC workflows.

Intermediate

Lesson 9 - Session & QuickNII Roundtrip

1

Save state during long work

Click Save Session in the top bar. This stores GUI state and makes long curation work resumable.

2

Load previous sessions

Click Load Session / QuickNII and select .deepslice-session.json to continue exactly where you left off.

3

Load QuickNII JSON/XML for re-curation

Use the same button to load existing QuickNII files. Species and markers are preserved when available.

✓ Outcome

You can pause/resume curation and roundtrip external QuickNII outputs without losing context.

Intermediate

Lesson 10 - Diagnostics, Error Reports, and Auto-Fix

This app includes structured logging and guided remediation for common runtime issues.

Open Error Log

Opens persistent log file for full traceback and runtime details.

Copy Last Error

Copies a complete report payload with context, traceback, and analysis.

Try Auto-Fix Last Error

Runs safe automated fixes (for known patterns like missing dependencies).

Error Dialog Buttons

Use Copy Error Report to share diagnostics; use Try Auto-Fix when available.

Recommended response sequence

  1. Read the dialog context and analysis summary.
  2. Click Try Auto-Fix if offered.
  3. If still failing, click Copy Error Report and share that payload.
  4. Open log file to inspect full traceback chronology.

✓ Outcome

Errors become actionable quickly, with machine-readable details suitable for AI-assisted repair.

Troubleshooting

ProblemResolution
ModuleNotFoundError: No module named 'tensorflow'Activate project venv and run python -m pip install "tensorflow<3.0".
ModuleNotFoundError: No module named 'skimage'Run python -m pip install scikit-image in the same activated project venv.
Run Alignment disabledRead Validation: text in Configuration and Ingestion warnings (common causes: duplicate/missing section numbers, non-coronal selection, too few slices).
No section numbers detectedRename files to include _sXXX pattern or enable legacy parser fallback.
Hardware Health shows CPU onlyExpected on many native Windows TensorFlow setups. CPU mode is supported.
QuickNII launch failsSet valid QuickNII executable path in Export panel and ensure JSON export exists.
Atlas preview unavailableCheck runtime log for atlas download/import issues and verify network/path permissions for metadata assets.
Prediction failed dialog appearsUse Copy Error Report and share the full payload; run Try Auto-Fix if available.
Do not ignore repeated prediction errors. If the same traceback repeats, capture one complete report and resolve root cause before re-running multiple times.
